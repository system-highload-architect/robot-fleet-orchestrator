# Риски и их митигация

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| **Потеря связи с роботом** (обрыв сети, севший аккумулятор) | Высокая | Критическое | Таймауты, повторные попытки (retry), перевод робота в состояние `offline`. Перераспределение задач на других роботов. Graceful degradation – система продолжает работу с оставшимися роботами. |
| **Перегрузка оркестратора** (слишком много запросов или роботов) | Средняя | Высокое | Rate limiting, горизонтальное масштабирование (множество экземпляров оркестратора), очереди запросов. Использование паттерна Circuit Breaker. |
| **Сложность распила на микросервисы** (модули слишком сильно связаны) | Средняя | Высокое | С самого начала строгое разделение по DDD, чёткие интерфейсы и событийная коммуникация. ADR для каждого решения. Регулярные архитектурные ревью. |
| **Unity-симуляция слишком ресурсоёмка** (тормозит, не успевает за реальным временем) | Средняя | Среднее | Использовать **DOTS (ECS + Burst Compiler + C# Job System)** для эффективной обработки тысяч объектов. Оптимизировать физику, использовать LOD (уровни детализации) и упрощённые модели для массовых объектов. При необходимости — запускать симуляцию без 3D-графики (headless mode) для нагрузочного тестирования. |
| **Задержки в распределённых транзакциях** (при распиле) | Средняя | Высокое | Использовать паттерн Saga для обеспечения консистентности. Идемпотентность операций. Event sourcing для восстановления состояния. |
| **Недостаточная производительность базы данных** (особенно ClickHouse) | Низкая | Среднее | Правильное индексирование, партиционирование, шардирование. Репликация. Использование кэша (Redis) для горячих данных. |
| **Безопасность** – подмена команд роботам, утечка данных | Низкая | Критическое | JWT-аутентификация, TLS, строгая валидация входных данных, аудит логов. Регулярное обновление зависимостей. |
| **Команда не успевает по срокам** | Средняя | Высокое | Приоритизация задач, MVP-first подход. Гибкое планирование спринтов. Найм дополнительных инженеров по мере роста. |
