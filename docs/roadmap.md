# Дорожная карта

## Этап 0 (1–2 месяца) – MVP

**Цель**: запустить работающий прототип, который может управлять парком эмулируемых роботов.

- Модульный монолит на Go с in-memory event bus.
- Эмулятор роботов (консольный, принимает команды через REST или каналы).
- PostgreSQL + Redis.
- Базовый оркестратор: выдача задач, отслеживание статусов.
- Простейший REST API для внешних систем.
- Docker Compose для локального окружения.

- **Управление**: формирование команды (поиск первых 2–3 разработчиков), настройка процессов, старт менторинга.

---

## Этап 1 (3–6 месяцев) – Расширение

**Цель**: перейти к реальным роботам, внедрить шину событий на базе Kafka и начать симуляцию в Unity.

- Интеграция с реальными роботами через MQTT/WebSockets.
- Замена in-memory шины на Kafka (подготовка к распилу).
- Внедрение ClickHouse для аналитики.
- Разработка Unity-симуляции (простые кубы-роботы, управление через WebSocket).
- **Внедрение AI-модуля**:
  - Интеграция с готовыми моделями (YOLO, Whisper).
  - Обработка изображений/звука в централизованном режиме.
  - Публикация событий распознавания.
- Автоматизированные тесты на симуляторе.

- **Управление**: расширение команды до 5–7 инженеров, внедрение code review и CI/CD, регулярные 1-1 с командой.

---

## Этап 2 (6–12 месяцев) – Масштабирование

**Цель**: распил на микросервисы, поддержка 1000+ роботов, полноценный цифровой двойник.

- Вынос модулей в отдельные сервисы: Robot Manager, Task Dispatcher, Location Service, Analytics, **AI Processor**.
- Kubernetes для оркестрации.
- Полноценная Unity-симуляция с физикой, сетевыми эмуляциями и симуляцией AI-сенсоров.
- Автоматическое масштабирование (HPA), мониторинг, алертинг.
- **Периферийный AI**: выполнение части моделей на самих роботах для снижения задержек.

- **Управление**: формирование подкоманд, делегирование ответственности, развитие лидеров внутри команды, активное участие в найме.

---

## Этап 3 (12+ месяцев) – Интеллектуализация

**Цель**: внедрение роевых алгоритмов, машинное обучение для оптимизации маршрутов и адаптивного поведения.

- Локальные агенты для автономного перераспределения задач при потере связи с центром.
- ML-модели для прогнозирования загрузки, профилактики сбоев.
- **Самообучение AI**: использование собранных данных для улучшения моделей распознавания.
- Расширенная аналитика, дашборды для менеджеров.
