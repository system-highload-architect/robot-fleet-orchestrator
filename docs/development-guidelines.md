# Стандарты разработки

## Code Style
- Использовать `golangci-lint` с рекомендованным набором линтеров.
- Соблюдать официальный стиль Go (gofmt, go vet).
- Комментарии к экспортируемым функциям и типам – обязательны.

## Git Flow
- Основные ветки: `main` (продакшен), `develop` (разработка).
- Фичи – в ветках `feature/xxx`, мержить через Pull Request с ревью.
- Коммиты – осмысленные, начинающиеся с глагола (Add, Fix, Update).

## Тестирование
- **Unit-тесты** – для каждой бизнес-логики (покрытие > 70%).
- **Интеграционные тесты** – для проверки взаимодействия модулей и БД (использовать testcontainers).
- **Нагрузочное тестирование** – с использованием `k6` или `vegeta`.

## Code Review
- Каждый PR проверяется как минимум одним разработчиком.
- Критерии: корректность функционирования и обработки исключений, читаемость, архитектурная чистота, наличие тестов, производительность.

## CI/CD
- Автоматическая сборка, тестирование, линтинг при каждом push в `develop`.
- Деплой в staging – автоматически, в production – по тегу.
- Использовать GitHub Actions или GitLab CI.
