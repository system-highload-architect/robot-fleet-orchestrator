# ADR-004: Включение AI-модуля в архитектуру

## Контекст
Роботы (особенно дроны) могут использоваться не только для доставки, но и для инспекции, поиска, взаимодействия с персоналом. Возможность распознавать объекты, выполнять голосовые команды и анализировать видео значительно расширяет функциональность системы и повышает её ценность для бизнеса.

## Рассмотренные варианты
- **Без AI** – система только управляет перемещением, без интеллектуальной обработки.
- **Встроенный AI-модуль** – выделенный модуль, работающий параллельно с оркестратором.

## Решение
Добавить модуль `AI Processor`, который подписывается на события и публикует результаты распознавания. Модуль может работать в двух режимах: централизованно (на бэкенде) и периферийно (на роботе). Это позволяет балансировать задержку и вычислительные затраты.

## Последствия
- **Плюсы**: расширение функционала, конкурентное преимущество, гибкость.
- **Минусы**: дополнительная сложность, нагрузка на инфраструктуру, потребность в AI-специалистах.
